- project trainee
	ในส่วนของ project trainee
	โปรเจคที่ได้รับมอบหมายให้รับผิดชอบคือ ระบบจัดการห้องสมุด หรือ Library Management System 
	
	เนื่องจากระบบเก่าไม่พร้อมใช้งาน ซึ่งการแก้ปัญหาชั่วคราวเป็นการใช้ excel ในการจัดการ
	ดังนั้นจึงได้ทำการสร้างระบบใหม่ขึ้นมาเพื่อทดแทนตัวระบบเก่า
	โดยใช้ภาษาคอมพิวเตอร์ในการพัฒนา ให้อยู่ในรูปแบบของ เว็บแอพพลิเคชัน
	โดยมีจุดประสงค์	- เพื่อมาทดแทนระบบเก่า
			- เพื่อการใช้งานที่สะดวกขึ้น
			- เพื่อเป็นระบบพื้นฐานในการจัดการห้องสมุด
	สิ่งที่คาดหวัง	- ผู้ใช้มีความพึงพอใจต่อการใช้งาน
			- มีความสะดวกต่อการใช้งาน
			- เป็นระบบพื้นฐานในการจัดการห้องสมุด
- user target
	ผู้ที่เกี่ยวข้องกับระบบมีด้วยกัน 2 ส่วน คือ
	ส่วนแรก admin ซึ่งหมายถึง ส่วนงาน Training
	มีหน้าที่ในการทำรายการยืม คืน ให้แก่ผู้ใช้งานห้องสมุด จัดการในส่วนของข้อมูลพื้นฐาน และนำออกรายงานต่าง ๆ ได้
	
	ส่วนที่ 2 user ซึ่งหมายถึง พนักงานทุก ๆ แผนก
	โดยสามารถใช้งานห้องสมุดได้ตามปกติ และสามารถดูประวัติการทำรายการของตนผ่านระบบได้โดยตรง
- current system
	ในส่วนของกระบวนการทำงาน ณ ปัจจุบัน การทำงานจะใช้โปรแกรม excel ในการจัดการ
	โดยการยืมหรือคืน พนักงานจะทำการนำหนังสือมาที่เค้าเตอร์ และเจ้าหน้าที่จะดำเนินการ
	โดย การยืม มีกระบวนการดังนี้ 	1. กรอกรหัสพนักงาน เลขเรียกหนังสือ ชื่อหนังสือ และวันที่ยืม ลง excel ซึ่งถ้ามีหลายรายการ ก็จะต้องทำแบบนี้วนไปเรื่อยๆ
			2. ปั้มวันที่ต้องคืนที่หลังหนังสือ 
	คือถ้ามีหลายรายการจะต้องทำแบบนี้วนไปเรื่อยๆ
	และการคืน 1. ปั้มวันที่ต้องคืนหลังหนังสือ
		2. ค้นหารายการใน excel และลบรายการนั้น ๆ
- new system
	และในส่วนของระบบใหม่ที่ได้พัฒนาขึ้นมา
	กระบวนการทำงานจะคล้ายกับกระบวนการปัจจุบัน แต่จะแตกต่างตรงที่เราได้นำระบบเข้ามาช่วยในการจัดการ
	โดยการยืม/คืน พนักงานจะนำหนังสือมาที่เค้าเตอร์ และเจ้าหน้าที่จะทำการใส่รหัสพนักงาน และแล้วสแกนบาร์โค้ด
	(การสแกนบาร์โค้ดสามารถสแกนได้ครั้งละมากว่า1เล่ม) และกดยืนยัน
	ซึ่งระบบจะเข้ามาช่วยในการจัดเก็บประวัติการทำรายการ รวมถึงข้อมูลรายการหนังสือ วารสาร โสตทัศนวัสดุ อย่างมีประสิทธิภาพ
	และมีการนำข้อมูลเหล่านี้มาสรุปผลในรูปแบบของรายงาน excel ได้อีกด้วย
- current system vs new system
	จากที่กล่าวไปข้างต้น 
	- ในด้านของขั้นตอนการทำงาน ถ้าเป็นการยืม จะเห็นได้ว่ากระบวนการเก่าจะต้องทำการกรอกรหัสพนักงาน เลขเรียกหนังสือ ชื่อหนังสือ และวันที่ยืม ต่อหนึ่งรายการ ซึ่งถ้ามีหลายรายการ ก็จะต้องทำแบบนี้วนไปเรื่อยๆ
				แต่ระบบใหม่แค่กรอกรหัสพนักงานในครั้งแรก และสแกนบาร์โค้ด สามารถสแกนได้ครั้งละหลายๆรายการ แล้วกดปุ่มยืนยันเป็นอันเสร็จ
	- ในด้านการเก็บข้อมูล การทำงานแบบเก่าจะลบรายการเมื่อมีการคืนหนังสือ แต่ระบบใหม่มีการเก็บข้อมูลของรายการนั้น ๆ
	- และสุดท้ายในด้านของรายงาน ระบบใหม่มีการเก็บข้อมูลในส่วนต่าง ๆ จึงสามารถมีการสรุปเป็นรายงานได้ แต่กระบวนการเก่าไม่สามารถทำได้หรือไม่ครบถ้วนนะครับ
